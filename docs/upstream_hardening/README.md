# Upstream Hardening Library

Purpose:

This directory preserves every significant defect inherited from upstream code and corrected in Stilus.

Goals:

- intellectual honesty
- engineering traceability
- regression prevention
- preservation of project knowledge

Every documented item should contain:

- origin
- discovery
- reproduction
- root cause
- patch
- validation
- remaining risks

A fix is not considered fully integrated into Stilus until it is documented here.
