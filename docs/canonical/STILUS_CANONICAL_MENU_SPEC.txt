STILUS CANONICAL MENU SPECIFICATION

Status: CANONICAL

Version: 1.0

Scope: User Interface / Menu System

======================================================================
PURPOSE
======================================================================

This document defines the canonical menu structure of Stilus.

The menu system is designed for:

- academic writing
- essays
- articles
- books
- homilies
- research-oriented writing
- long-form text production

Stilus remains:

- lightweight
- native desktop
- text-oriented
- TXT-first
- Markdown-friendly

Stilus is not:

- an IDE
- a PKM
- a knowledge base
- a workspace manager
- a project manager
- a Scrivener clone
- an Obsidian clone

======================================================================
RESEARCH PANEL
======================================================================

The right-side panel is canonically named:

Research Panel

The Research Panel contains:

- Research Clipboard
- Tags Collection
- Source Notes
- Scratchpad
- Document Outline

Important distinction:

Document Outline is visually present inside the Research Panel.

Document Outline conceptually belongs to Navigate.

The Research Panel must remain:

- lightweight
- text-oriented
- non-database
- non-PKM
- non-workspace

Forbidden concepts:

- Graph View
- Backlinks
- Vault
- Workspace
- Knowledge Base
- Second Brain
- Semantic Engine
- AI Assistant

======================================================================
MULTIPLE WINDOWS POLICY
======================================================================

Stilus supports multiple independent windows.

Canonical workflow:

Window 1 вҶ’ Manuscript
Window 2 вҶ’ Citations
Window 3 вҶ’ References
Window 4 вҶ’ Notes

Multiple independent windows are preferred over split-view complexity.

======================================================================
IMPLEMENTATION DISCIPLINE
======================================================================

This menu specification is canonical.

Implementation must remain consistent with:

- lightweight architecture
- academic writing workflow
- research-assisted writing
- TXT-first philosophy
- Markdown-friendly philosophy

No menu item authorizes:

- IDE features
- PKM features
- workspace systems
- project management systems
- knowledge graph systems
- cloud services
- AI integrations
- architectural rewrites
